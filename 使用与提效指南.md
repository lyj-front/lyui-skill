## 一、项目简介

### 1.1 什么是 LyUI Skill

LyUI Skill 是一个为 Cursor/AI 编程助手设计的组件库技能包，提供：

- **智能组件检索**：基于倒排索引的高效搜索
- **拼音搜索支持**：支持中文拼音输入（如 "anniu" 匹配 "按钮"）
- **文档索引管理**：自动同步和索引组件文档
- **代码示例推荐**：根据使用场景智能推荐组件

### 1.2 核心优势

| 特性 | 说明 | 性能 |
|------|------|------|
| 倒排索引 | O(1) 关键词查找 | 0.000ms/op |
| 拼音搜索 | 无需切换输入法 | 0.101ms/op |
| 模糊匹配 | n-gram 相似度计算 | 0.071ms/op |
| 场景推荐 | 基于关键词权重匹配 | < 1ms |

### 1.3 适用场景

- ✅ AI 编程助手集成（Cursor、Windsurf 等）
- ✅ 中文开发团队
- ✅ 大型组件库管理
- ✅ 快速组件查找和代码生成

## 三、核心功能使用

### 3.1 组件搜索

#### 基础搜索

```typescript
import { searchComponents } from './src/index.js';

// 中文搜索
const buttons = searchComponents('按钮');
// 返回: [button, button-group, ...]

// 英文搜索
const tables = searchComponents('table');
// 返回: [table, table-page, ...]

// 拼音搜索
const results = searchComponents('anniu');
// 返回: [button] ("按钮"的拼音)
```

#### 高级搜索选项

```typescript
import { searchComponents } from './src/index.js';

const results = searchComponents('table', {
  limit: 5,        // 最多返回5个结果
  fuzzy: true,     // 启用模糊匹配
  threshold: 0.3   // 最低匹配分数
});
```

#### 场景推荐

```typescript
import { suggestComponents } from './src/index.js';

// 根据使用场景推荐组件
const suggestions = suggestComponents('表单提交', 5);
// 返回:
// [
//   { component: form, relevance: 0.95 },
//   { component: button, relevance: 0.85 },
//   { component: input, relevance: 0.80 },
//   ...
// ]
```

### 3.2 组件元数据查询

```typescript
import { 
  getComponentMeta, 
  getRelatedComponents,
  getComponentsByCategory 
} from './src/index.js';

// 获取组件元数据
const buttonMeta = getComponentMeta('button');
// 返回:
// {
//   id: 'button',
//   displayName: 'Button',
//   category: 'basic',
//   keywords: ['button', 'btn', '按钮'],
//   related: ['button-group', 'link'],
//   complexity: 'simple',
//   docPath: 'docs/button.md'
// }

// 获取相关组件
const related = getRelatedComponents('button');
// 返回: [button-group, link]

// 按分类获取组件
const basicComponents = getComponentsByCategory('basic');
// 返回: [button, icon, link, ...]
```

### 3.3 文档索引

```typescript
import { 
  listDocFiles, 
  hasDocFile, 
  getMetaByDocFile 
} from './src/index.js';

// 列出所有文档
const docs = listDocFiles();
// 返回: ['button.md', 'table.md', ...]

// 返回: button 组件的元数据
```

### 3.4 分类统计

## 四、开发工作流

### 4.1 文档同步

```bash
# 从上游同步文档
npm run sync-docs

# 生成文档索引
npm run generate-doc-index
```

### 4.2 组件开发

#### 生成新组件

```bash
# 使用组件生成器
npm run generate-component -- --name my-component --category form --display-name "我的组件"

# 参数说明:
# --name: 组件ID (必需)
# --category: 分类 (basic/form/data/feedback/navigation/business/guide/other)
# --display-name: 显示名称
```

生成后需要：
1. 在 `src/core/component-registry.ts` 中添加注册表条目
2. 编辑 `docs/my-component.md` 完善文档
3. 运行 `npm run sync-docs` 同步

#### 组件注册表示例

```typescript
// src/core/component-registry.ts
{
  id: 'my-component',
  displayName: '我的组件',
  category: 'form',
  keywords: ['my-component', 'ly-my-component', '我的组件'],
  related: ['input', 'form'],
  complexity: 'simple',
  docPath: 'docs/my-component.md',
}
```

### 5.1 搜索技巧

| 搜索方式 | 示例 | 说明 |
|----------|------|------|
| 精确ID | `button` | 最快，O(1)查找 |
| 中文 | `按钮` | 支持中文关键词 |
| 拼音 | `anniu` | 无需切换输入法 |
| 模糊 | `but` | 前缀匹配 |
| 场景 | `表单提交` | 智能推荐 |

### 5.2 快捷键（Cursor 集成）

```
# 在 Cursor 中使用
@lyui 按钮          # 搜索按钮组件
@lyui table page    # 搜索表格分页
@lyui 表单          # 搜索表单相关组件
```

### 5.3 批量操作脚本

```bash
# 分析组件使用情况
npm run analyze-usage

# 生成完整文档索引
npm run generate-doc-index
```

### 5.4 性能优化建议

1. **搜索缓存**：结果自动缓存，重复查询更快
2. **索引预构建**：启动时构建，避免运行时重建
3. **按需加载**：使用特定导入减少加载时间

```typescript
// 推荐：按需导入
import { searchComponents } from './src/core/registry.js';

// 不推荐：全量导入（除非需要所有功能）
import * as lyui from './src/index.js';
```

## 六、常见问题

### Q1: 拼音搜索不准确？

**A**: 当前使用预定义词典，可扩展 `src/core/pinyin.ts` 中的 `PINYIN_DICT`：

```typescript
const PINYIN_DICT: Record<string, string> = {
  '你的词汇': 'nidecihui',
  // ...
};
```

### Q2: 如何添加新的组件分类？

**A**: 
1. 在 `src/core/types.ts` 的 `ComponentCategory` 中添加
2. 在 `src/core/component-registry.ts` 的 `CATEGORY_NAMES` 中添加显示名
3. 更新分类分组逻辑

### Q3: 搜索返回空结果？

**A**: 检查以下几点：
- 组件是否已注册
- 文档是否已同步
- 搜索关键词是否正确
- 是否使用了不支持的分类

### Q4: 如何调试搜索算法？

**A**: 使用 `SearchIndex` 类直接测试：

```typescript
import { SearchIndex } from './src/core/search-index.js';
import { COMPONENT_REGISTRY } from './src/core/component-registry.js';

const index = new SearchIndex(COMPONENT_REGISTRY);
const results = index.search('按钮', { limit: 10, fuzzy: true });
console.log(results);
```

### Q5: CI/CD 失败？

**A**: 本地先运行完整检查：

```bash
npm run ci
```

确保：
- 所有测试通过
- 无 ESLint 错误
- TypeScript 类型检查通过
